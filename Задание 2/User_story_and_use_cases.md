## User Story:
Я, как Продавец, хочу иметь возможность опубликовать свой товар на маркетплейсе (витрине товаров).

## Критерии приёмки и ограничения:
#### Карточка товара должна содержать:
1. Наименование товара
2. Фото/видео-обзор товара
3. Описание товара
4. Характеристики товара
5. Стоимость

#### Ограничения:
1. Фото/видео-обзор должны проходить возрастной ценз
2. Загружаемые материалы должны быть разрешенного формата
3. Нельзя завести несколько разных товаров на одной карточке
4. Нельзя создать две карточки товаров с полностью идентичным названием
5. Стоимость не может быть равна нулю или отрицательной

## Use Case:
| Параметр | Описание |
| :--- | :--- |
| **Вариант использования** | Добавить товар на маркетплейс (витрину товаров) |
| **Основное действующее лицо** | Любой зарегистрированный пользователь-продавец данного интернет-магазина |
| **Предусловие** | Пользователь-продавец авторизован на сайте, находится в личном кабинете |
| **Триггер** | Пользователь-продавец нажимает кнопку «Добавить товар» в личном кабинете |
| **Гарантии успеха** | Карточка товара успешно добавлена на маркетплейс |

#### Описание
1. Пользователь-продавец заполняет обязательные и необязательные (по желанию) текстовые поля карточки товара.
2. Пользователь-продавец загружает фото/видео-обзор товара.
	1. При попытке загрузить неверный формат, система не загружает их, сообщает об ошибке и дает пользователю-продавцу список из разрешенных форматов.
	2. Пользователь-продавец загружает файлы разрешенных форматов.
3. Система проверяет введенные данные на соответствие ограничениям полей формы и на уникальность.
4. Система подтверждает возможность добавления товара в профиль пользователя-продавца и появляется предпросмотр карточки товара (как оно будет выглядеть на маркетплейсе).
5. Пользователь-продавец подтверждает создание карточки товара.
6. Система добавляет карточку товара в профиль пользователя-продавца в соответствии с введенными данными.
  
#### Расширение
1. Пользователь-продавец получает сообщения о некорректно заполненных полях и предложение отредактировать их.
2. Пользователь-продавец вносит изменения в соответствующие поля.
3. Система возвращается к шагу 6.3.
4. Системная ошибка: при возникновении внутреннего сбоя (нет связи с БД, ошибка на сервере):
	1. При возникновении ошибки на сервере система возвращается в состояние 3 (проверка на соответствие и уникальность).
	2. При возникновении ошибки в БД система возвращается в состояние 0 - все поля формы пустые. Пользователь-продавец возвращается к пункту 1 (заполнение обязательных и необязательных текстовых полей).
