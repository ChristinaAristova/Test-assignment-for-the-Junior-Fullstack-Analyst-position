## Алгоритм создания пользователя (Backend):
1. Прием запроса: Сервер получает POST запрос на /register с телом в формате JSON.
2. Проверка наличия обязательных полей:
    - Если отсутствует firstName, lastName, userName, password или recaptchaToken -> сразу возврат ошибки 400 Bad Request с описанием "Missing required fields".
3. Валидация полей:
    - Проверить recaptchaToken через Google API. Если недействителен -> возврат 400 Bad Request "Please verify reCaptcha to register!".
    - Проверить password на соответствие требованиям (длина, символы). Если не проходит -> запомнить ошибку "Passwords must have at least one non alphanumeric character, one digit ('0'-'9'), one uppercase ('A'-'Z'), one lowercase ('a'-'z'), one special character and Password must be eight characters or longer.".
    - Проверить firstName, lastName на допустимые символы (если правило есть). Если не проходят -> запомнить ошибку.
4. Проверка уникальности:
    - Выполнить запрос в БД: SELECT id FROM users WHERE userName = :userName.
    - Если запись найдена -> запомнить ошибку "User exists!".
5. Формирование ответа:
    - Если список ошибок не пуст -> вернуть 400 Bad Request (или 409 Conflict, если единственная ошибка - конфликт имен), перечислив все найденные ошибки в теле ответа.
    - Если ошибок нет -> продолжить.
6. Сохранение в БД:
    - Захэшировать пароль (например, bcrypt).
    - Выполнить INSERT INTO users (firstName, lastName, userName, password_hash) VALUES (...).
    - Если возникает ошибка БД -> возврат 500 Internal Server Error.
7. Успешный ответ:
    - Получить id созданной записи (через lastInsertId или аналогичный метод).
    - Вернуть 201 Created с JSON, содержащим id, firstName, lastName, userName.


#### Примечание
В пунктах 2 и 6 конкретные тексты ошибок для отсутствующих полей и внутренней ошибки сервера не были предоставлены во вводных данных. В пункте 3 при возникновении ошибки заполнения полей First Name и Last Name некорректными данными отсутствует текст ошибки, поскольку требования к этим полям отсутствуют. Я включила их как логически необходимые шаги, но на реальном проекте я бы запросила у разработчиков точные формулировки.
